# AURA 앱 — 상담 챗(Consultation Chat) 기능 추가 명세서
> 버전: 1.0 | 기반: 아우라앱_기술명세서 v1.0 | 신규 기능 단일 명세

---

## ◆ 이 문서의 목적

기존 Aura 앱은 **관상/수상 분석 → 결과 출력 → 저장**까지 동작 중이다.
이 명세서는 그 위에 **상담 챗(Consultation Chat)** 기능을 추가하기 위한 작업 지시서다.

신규 기능 한 줄 정의:
> "관상·수상 분석 결과를 컨텍스트로, Gemma와 멀티턴 대화를 이어가며 해석을 심화 질문할 수 있는 챗 기능"

---

## ◆ 에이전트 작업 분담 (기존 5에이전트 구조 재사용)

```
AGENT_1 (architect)  — 라우트 추가, 폴더 추가, 신규 Provider 등록
AGENT_2 (ai_engine)  — Gemma 챗 세션 관리, 컨텍스트 윈도잉, 토큰 예산 관리
AGENT_3 (ui_screens) — 상담 목록 / 분석 선택 / 챗 화면 3개 구현
AGENT_4 (backend)    — consultations / consultation_messages 테이블, RLS, Realtime
AGENT_5 (i18n_qa)    — 4개 언어 ARB 키 추가, 다크모드 검증, E2E 테스트
```

### 실행 순서
```
1단계(직렬): AGENT_4 (DB 스키마) → AGENT_1 (라우트/폴더)
2단계(병렬): AGENT_2 (Gemma 세션) + AGENT_3 (UI 3화면)
3단계(직렬): AGENT_5 (i18n + QA)
```

### 공통 규칙
- 본 문서 전체를 작업 시작 전 반드시 정독한다
- 기존 `아우라앱_기술명세서`와 충돌이 발생하면 본 문서가 우선한다
- 단, **flutter_gemma 버전은 실제 사용 중인 `0.13.2`** 기준으로 작성됨에 유의한다
- 커밋 메시지: `[AGENT_N][CONSULT] 작업내용`

## 1. 기능 요구사항 요약

| 항목 | 내용 |
|------|------|
| 진입 경로 1 | **결과 화면 → "상담하기" 버튼** (방금 본 분석 자동 컨텍스트화) |
| 진입 경로 2 | **메인 → "상담하기" 메뉴 → 분석 선택 → 챗 진입** (저장된 분석 중 선택) |
| 인증 | **로그인 필수** (비로그인 시 메인 메뉴 숨김 + 결과 화면 버튼 비활성+로그인 유도) |
| 범위 | 관상·수상 해석 심화 질문만 (Off-topic 질문은 부드럽게 거절) |
| 저장 | Supabase 영구 저장 (대화 전부) |
| 출력 | Gemma 스트리밍 출력 (typing 인디케이터 + 토큰 단위 append) |
| 다국어 | 한/영/중/일 4개 언어 모두 지원 |
| 다크모드 | 라이트/다크 모두 지원 |

## 2. 데이터베이스 설계 (AGENT_4)

### 2-1. 테이블 추가

```sql
-- 상담 세션 (한 분석에 대해 여러 상담 가능)
CREATE TABLE public.consultations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,

  -- 어떤 분석에서 시작됐는가 (다형성: face 또는 palm)
  analysis_type TEXT NOT NULL CHECK (analysis_type IN ('face', 'palm')),
  analysis_id UUID NOT NULL,  -- face_analyses.id 또는 palm_analyses.id

  -- 챗에 주입할 압축 컨텍스트 (생성 시 1회 고정)
  context_summary TEXT NOT NULL,  -- 분석 결과 요약 (최대 800자)
  context_features JSONB NOT NULL, -- 구조화된 특징 (faceShape, eyeType, ratios 등)

  -- 표시용 메타
  title TEXT,                      -- "오늘의 관상 상담" 등 (자동 생성 또는 사용자 편집)
  locale TEXT NOT NULL DEFAULT 'ko',
  model_used TEXT NOT NULL,        -- "e2b" | "e4b" | "b4b"
  message_count INT NOT NULL DEFAULT 0,
  last_message_at TIMESTAMPTZ DEFAULT NOW(),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 상담 메시지 (각 턴)
CREATE TABLE public.consultation_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  consultation_id UUID NOT NULL REFERENCES consultations(id) ON DELETE CASCADE,
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
  content TEXT NOT NULL,
  token_count INT,                 -- 추후 윈도잉 계산용 (선택)
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 2-2. 인덱스

```sql
CREATE INDEX consultations_user_id_idx
  ON consultations(user_id, last_message_at DESC);

CREATE INDEX consultations_analysis_idx
  ON consultations(analysis_type, analysis_id);

CREATE INDEX consultation_messages_consultation_id_idx
  ON consultation_messages(consultation_id, created_at ASC);
```

### 2-3. Row Level Security

```sql
ALTER TABLE consultations ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users can insert own consultations"
  ON consultations FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "users can view own consultations"
  ON consultations FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "users can update own consultations"
  ON consultations FOR UPDATE USING (auth.uid() = user_id);
CREATE POLICY "users can delete own consultations"
  ON consultations FOR DELETE USING (auth.uid() = user_id);

ALTER TABLE consultation_messages ENABLE ROW LEVEL SECURITY;

-- 메시지는 부모 consultation의 소유자만 접근 가능
CREATE POLICY "users can insert messages to own consultations"
  ON consultation_messages FOR INSERT WITH CHECK (
    EXISTS (
      SELECT 1 FROM consultations c
      WHERE c.id = consultation_id AND c.user_id = auth.uid()
    )
  );
CREATE POLICY "users can view messages of own consultations"
  ON consultation_messages FOR SELECT USING (
    EXISTS (
      SELECT 1 FROM consultations c
      WHERE c.id = consultation_id AND c.user_id = auth.uid()
    )
  );
CREATE POLICY "users can delete messages of own consultations"
  ON consultation_messages FOR DELETE USING (
    EXISTS (
      SELECT 1 FROM consultations c
      WHERE c.id = consultation_id AND c.user_id = auth.uid()
    )
  );
```

### 2-4. 트리거 (메시지 카운트/시간 자동 갱신)

```sql
CREATE OR REPLACE FUNCTION update_consultation_meta()
RETURNS TRIGGER AS $$
BEGIN
  UPDATE consultations
  SET message_count = message_count + 1,
      last_message_at = NEW.created_at
  WHERE id = NEW.consultation_id;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER consultation_message_inserted
  AFTER INSERT ON consultation_messages
  FOR EACH ROW EXECUTE FUNCTION update_consultation_meta();
```

### 6-4. 컨텍스트 요약 생성 전략

`consultation.context_summary` 는 **상담 생성 시 1회 고정**한다.

**옵션 A (권장):** 기존 `face_analyses.result_text` 의 첫 600자를 그대로 사용
- 장점: 별도 Gemma 호출 없음, 빠름
- 단점: 컷오프가 어색할 수 있음

**옵션 B:** Gemma로 요약 생성 (200~300자)
- 장점: 깔끔한 요약
- 단점: 상담 생성에 5~15초 추가 소요

→ **옵션 A로 시작하고, 사용자 피드백 후 B로 업그레이드**

```dart
String buildContextSummary(String fullResultText) {
  if (fullResultText.length <= 600) return fullResultText;
  return '${fullResultText.substring(0, 600)}…';
}
```

### 6-5. 챗 세션 라이프사이클

```dart
// lib/features/consultation/providers/gemma_chat_session_provider.dart

class GemmaChatSessionNotifier extends AsyncNotifier<GemmaChat> {
  GemmaChat? _chat;
  String? _currentConsultationId;

  /// 화면 진입 시 호출
  /// - 새 consultation: systemInstruction만 설정한 fresh chat 생성
  /// - 기존 consultation: 메시지 로드 → sliding window 적용 → replay로 컨텍스트 복원
  Future<void> openSession(Consultation consultation) async {
    // 이미 같은 세션이면 재사용
    if (_currentConsultationId == consultation.id && _chat != null) return;

    // 이전 세션 정리
    await _chat?.close();

    final model = await FlutterGemma.getActiveModel(maxTokens: 2048);
    _chat = await model.createChat(
      modelType: ModelType.gemmaIt,
      systemInstruction: buildConsultationSystemInstruction(
        locale: consultation.locale,
        analysisType: consultation.analysisType,
        contextSummary: consultation.contextSummary,
        features: consultation.contextFeatures,
      ),
      temperature: 0.7,
      topK: 40,
    );

    // DB에서 메시지 로드 후 윈도잉 + replay
    final messages = await ref
        .read(consultationServiceProvider)
        .loadMessages(consultation.id);
    final windowed = _applySlidingWindow(messages);
    for (final msg in windowed) {
      await _chat!.addQueryChunk(
        Message(text: msg.content, isUser: msg.role == MessageRole.user),
      );
    }

    _currentConsultationId = consultation.id;
    state = AsyncData(_chat!);
  }

  /// 사용자 메시지 전송 → 스트림으로 응답 토큰 방출
  Stream<String> sendMessage(String userInput) async* {
    assert(_chat != null, 'Chat session not opened');
    await _chat!.addQueryChunk(Message(text: userInput, isUser: true));
    await for (final chunk in _chat!.generateChatResponseAsync()) {
      yield chunk.text;  // ModelResponse.text
    }
  }

  /// 최근 N턴(혹은 토큰 합 기준)만 유지하는 슬라이딩 윈도우
  List<ConsultationMessage> _applySlidingWindow(
    List<ConsultationMessage> all, {
    int maxTurns = 12,  // 6 user + 6 assistant 정도
  }) {
    if (all.length <= maxTurns) return all;
    return all.sublist(all.length - maxTurns);
  }

  Future<void> close() async {
    await _chat?.close();
    _chat = null;
    _currentConsultationId = null;
  }
}
```

### 6-6. 주의사항 (AGENT_2가 반드시 지킬 것)

- 화면 dispose 시 `chat.close()` 호출 (메모리 누수 방지)
- 사용자가 빠르게 연속 전송하지 못하도록 **전송 중에는 입력 비활성화**
- 스트리밍 중 사용자가 화면 이탈 시 응답을 끝까지 받아 DB에 저장한 뒤 세션 종료
- Gemma 응답이 비어있거나 실패 시 사용자 메시지는 롤백하지 않고 **에러 메시지를 assistant 메시지로 저장** ("응답 생성에 실패했습니다. 다시 시도해주세요.")

```dart
  final SupabaseClient _supabase;
  ConsultationService(this._supabase);

  /// 새 상담 생성 (결과 화면 또는 분석 선택 화면에서 호출)
  Future<Consultation> create({
    required String userId,
    required AnalysisType analysisType,
    required String analysisId,
    required String fullResultText,
    required Map<String, dynamic> features,
    required String locale,
    required String modelUsed,
  }) async {
    final summary = buildContextSummary(fullResultText);
    final autoTitle = _generateTitle(analysisType, locale);

    final response = await _supabase.from('consultations').insert({
      'user_id': userId,
      'analysis_type': analysisType.name,
      'analysis_id': analysisId,
      'context_summary': summary,
      'context_features': features,
      'title': autoTitle,
      'locale': locale,
      'model_used': modelUsed,
    }).select().single();

    return Consultation.fromJson(response);
  }

  /// 사용자별 상담 목록 (최근 활동 순)
  Future<List<Consultation>> listByUser(String userId) async {
    final rows = await _supabase
        .from('consultations')
        .select()
        .eq('user_id', userId)
        .order('last_message_at', ascending: false);
    return rows.map((r) => Consultation.fromJson(r)).toList();
  }

  /// 단건 조회
  Future<Consultation> get(String id) async {
    final row = await _supabase
        .from('consultations')
        .select()
        .eq('id', id)
        .single();
    return Consultation.fromJson(row);
  }

  /// 메시지 전체 로드 (오래된 순)
  Future<List<ConsultationMessage>> loadMessages(String consultationId) async {
    final rows = await _supabase
        .from('consultation_messages')
        .select()
        .eq('consultation_id', consultationId)
        .order('created_at', ascending: true);
    return rows.map((r) => ConsultationMessage.fromJson(r)).toList();
  }

  /// 메시지 1건 저장
  Future<ConsultationMessage> appendMessage({
    required String consultationId,
    required MessageRole role,
    required String content,
  }) async {
    final row = await _supabase.from('consultation_messages').insert({
      'consultation_id': consultationId,
      'role': role.name,
      'content': content,
    }).select().single();
    return ConsultationMessage.fromJson(row);
  }

  /// 상담 삭제 (CASCADE로 메시지 자동 삭제)
  Future<void> delete(String id) async {
    await _supabase.from('consultations').delete().eq('id', id);
  }

  String _generateTitle(AnalysisType type, String locale) {
    final dateStr = DateFormat('M.d').format(DateTime.now());
    return {
      'ko': type == AnalysisType.face ? '$dateStr 관상 상담' : '$dateStr 손금 상담',
      'en': type == AnalysisType.face
          ? '$dateStr Face Reading Consultation'
          : '$dateStr Palm Reading Consultation',
      'zh': type == AnalysisType.face ? '$dateStr 面相咨询' : '$dateStr 手相咨询',
      'ja': type == AnalysisType.face ? '$dateStr 観相相談' : '$dateStr 手相相談',
    }[locale]!;
  }
}
```
